Knowledge base (V0)

Purpose
- Capture the evolving trading system in plain English.
- Keep decisions, rules, and setup definitions in one place.
- Make iteration explicit and reviewable.

Structure
- setups/: one file per setup definition
- reviews/: weekly review snapshots
- daily reports: one per day in knowledge/reviews
- rules.md: risk policy and approval checklist
- go_live.md: measurable scorecard for live readiness
- changelog.md: what changed and why
- study.md: prompts for quiet days
- options_basics.md: options reference (learning only)

Use
- Update setup docs when rules change.
- Add a review snapshot weekly.
- Log system changes in changelog.md.
